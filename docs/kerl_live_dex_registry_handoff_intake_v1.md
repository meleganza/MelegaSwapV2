---

## 1. Registry purpose

Phase 7A introduces a **local registry handoff intake** inside Melega DEX. Certified dry-run KERL handoff packages are read from static JSON artifacts under `public/registry/kerl/`, validated, and forwarded through the existing **Certified Dry-Run Handshake** into the **Execution Gateway** (`DRY_RUN_ONLY`).

This is the first repository-aware intake path. It uses **filesystem reads only** — no `fetch`, no Swarm runtime, no wallet, no adapter dispatch.

## 5. Rejection rules

### Registry layer

| Condition | Error code |
|-----------|------------|
| Invalid JSON | `REGISTRY_MALFORMED_JSON` |
| Path traversal (`..`, absolute paths) | `REGISTRY_PATH_TRAVERSAL` |
| Network URL as path or in JSON strings | `REGISTRY_NETWORK_URL_FORBIDDEN` |
| Missing artifact file | `REGISTRY_ARTIFACT_NOT_FOUND` |
| Non-object root | `REGISTRY_NOT_OBJECT` |

### Handoff + handshake layer (delegated)

| Condition | Error code (representative) |
|-----------|----------------------------|
| `handoffMode: live` | `HANDOFF_LIVE_MODE_REJECTED` |
| Wallet data / interaction | `HANDOFF_WALLET_INTERACTION_IMPLIED` / handshake envelope errors |
| Transaction hash | `HANDOFF_TRANSACTION_HASH_IMPLIED` / `HANDSHAKE_INVALID_MANIFEST` |
| Receipt / settlement | `HANDOFF_RECEIPT_IMPLIED` / `HANDOFF_SETTLEMENT_IMPLIED` / `HANDSHAKE_INVALID_MANIFEST` |
| `executionPerformed: true` | `HANDOFF_EXECUTION_IMPLIED` / `HANDSHAKE_INVALID_MANIFEST` |
| Missing certification | `HANDSHAKE_MISSING_CERTIFICATION` |
| Invalid / incompatible certification | `HANDSHAKE_INVALID_CERTIFICATION_VERDICT` / `HANDSHAKE_CERTIFICATION_INCOMPATIBLE` |

Rejection always occurs **before** adapter dispatch and wallet submission.

## 6. Dry-run guarantees

| Guarantee | Enforced by |
|-----------|-------------|
| `executionMode: DRY_RUN_ONLY` | Manifest + gateway |
| `executionPerformed: false` | Manifest + gateway result |
| `walletInteraction: none` | Manifest + gateway result |
| `networkCommunication: false` | Registry index + manifest |
| `transactionHash: null` | Manifest + gateway result |
| `receipt: null` | Manifest + gateway result |
| `settlement: null` | Manifest + gateway result |
| No `fetch` / remote URLs in registry | `validateRegistryHandoffJson` + path guards |
| No adapter dispatch | Gateway boundary + intake tests |
| No UI / public API wiring | Surface isolation tests |
| No Swarm runtime import | Ownership constants |

## 8. Future Swarm publication path

When Swarm CI stabilizes:

1. Swarm publishes certified `DryRunHandoffPackage` JSON to `public/registry/kerl/handoffs/` via PR (or documented artifact drop).
2. Update `public/registry/kerl/index.json` with new `id`, `path`, version fields.
3. DEX CI runs `intakeRegistryHandoffFromFile()` against Swarm-published artifacts — **no Swarm runtime import**.
4. Version alignment enforced by existing `HANDSHAKE_VERSION_MISMATCH` gates.
5. Live handoff mode, network endpoints, and adapter dispatch remain **out of scope** until a future constitutional phase with explicit gates.
